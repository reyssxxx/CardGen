Привет. Мы делаем проект: телеграм-бот для информирования учеников конкретного лицея. Некоторая часть бота уже готова. Я опишу готовую часть, структуру и целевой функционал нашего бота, а ты поможешь в реализации. 

1. Целевой функционал:
Основная функция бота - иметь обновляемый список учеников и автоматически рассылать сообщения и файлы, которые загружают учителя. Рассылки производятся согласно классам, в которых учатся ученики. Главная фича бота - генерация табелей текущей успеваемости по фотографиям журнала, которые загружают учителя. А именно порядок обработки такой: получить фотографию, оцифровать ее, добавить оценки в электронное хранилище. Фотографии будут содержать шаблонный печатный лист, учителя делают фотографии самостоятельно, следовательно на вход будут попадать изображения под разными углами, с разными разрешениями и разным освещением. Имей это в виду. После загрузки фотографии учитель должен иметь возможность проверить и отредактировать результаты распознавания оценок. Раз в 2 недели бот должен будет создавать фотографии с оценками (табели) каждого конкретного ученика за период и рассылать каждому ученику его табель.
Мы с командой предусмотрели 3 уровня доступа пользователей: администратор, учитель, ученик.

Функционал администратора:
В идеале функционал администратора должен быть доступен в telegram. Если нет возможности реализовать какой-либо из этих пунктов удобным образом внутри бота telegram, предложи другую реализацию.
-обновлять список учителей
-загружать расписание (мгновенно рассылается всем пользователям)
-обновлять список учеников (ученики могут как зачисляться на обучение, так и отчисляться в течение года)
-просматривать текущую успеваемость учеников
-просматривать состояние журнала (все ли учителя загрузили фотографии к сроку)
-редактировать список предметов, их названия и любые другие данные в БД
-отправлять сообщения конкретным классам
-загружать фото журнала

Функционал учителя:
-загружать фото журнала
-отправлять сообщения для рассылки конкретным классам

Функционал ученика:
-просмотреть последний актуальный табель
-просмотреть текущие полугодовые и годовые оценки
-просмотреть текущую статистику по оценкам

2. Готовая часть:
У нас уже есть основа телеграм-бота, функции для регистрации учеников и учителей. На данный момент система следующая: telegram_id администраторов записаны в файле окружения .env, бот всегда знает, если ему пишет администратор. Функционал администратора не реализован вообще. Все файлы с учениками добавлены вручную. Когда новый пользователь пытается войти в систему через бота, он вводит свое ФИО и указывает, ученик он или учитель.  Выбор роли нужен, потому что для учителей и учеников разные проверки на доступ. Данные учеников сверяют с актуальным списком, который будет загружать администратор. Данные учителей так же сверяют со списком, но список учителей должен содержать расширенные данные: текущий username (чтобы ученики не представлялись учителями) и класс, если учитель - классный руководитель. Несколько функций по работе с БД так же реализовано, но это несущественно.  Так же уже реализована генерация изображений по шаблонам, тебе лишь нужно использовать эту функцию и корректно передавать ей оценки 

3. Структура
Мы ждем от тебя дописанный проект, содержащий следующие элементы:
-доработанная система регистрации и аутентификации пользователей (если считаещь нужным)
-система рассылки файлов и сообщений пользователям (как единоразовые, так и раз в 2 недели)
-функции для БД, полностью напиши с 0, можешь сам придумать и предложить структуру БД
-Важно: ты должен реализовать надежный алгоритм извлечения оценок из фотографий (описание процесса было в пункте с функционалом)
-весь указанный функционал пользователей
-описание структуры файла с учениками и учителями (или 2 раздельных файла). Требование: ФИО учеников должны быть разбиты по классам, к учителям должны быть прикреплены их username и ведомый класс (если учитель - классный руководитель)
