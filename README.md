# CardGen — Telegram-бот для лицея ЮФУ

Бот автоматизирует учёт оценок и коммуникацию между учениками, учителями, психологом и администрацией лицея Южного федерального университета.

## Что делает бот

### Для учеников
- **Табель оценок** — персональная сводная таблица оценок по всем предметам за двухнедельные периоды, доступная в любой момент прямо в Telegram
- **Мероприятия** — просмотр предстоящих событий лицея, запись на мероприятия и отдельные секции, управление своими записями
- **Объявления** — лента объявлений от администрации и учителей
- **Вопросы к администрации** — анонимные вопросы с возможностью получить ответ прямо в боте
- **Психологическая поддержка** — анонимный чат с психологом: можно начать разговор не раскрывая имени и раскрыть личность по желанию
- **Хендбук лицеиста** — справочник по жизни в лицее

### Для учителей
- **Рассылка объявлений** — отправка текста или фото конкретному классу из своего списка

### Для администраторов
- **Загрузка оценок** — импорт оценок из Excel-файла с предпросмотром и подтверждением
- **Шаблон Excel** — генерация пустого шаблона под нужный класс
- **Рассылка табелей** — ручная отправка персональных карточек оценок ученикам (или автоматическая раз в 2 недели)
- **Управление оценками** — просмотр и удаление оценок по конкретному ученику
- **Мероприятия** — создание событий с секциями, управление записями
- **Объявления** — рассылка всем или конкретному классу, с поддержкой фото
- **Q&A** — просмотр анонимных вопросов от учеников (с информацией об авторе) и ответ на них

### Для психолога
- **Чат с учениками** — просмотр активных анонимных чатов, ответы на сообщения, завершение диалогов

---

## Зачем это нужно

**Проблема:** В лицее не было единого канала для оценок, объявлений и обратной связи. Учителя рассылали оценки вручную или через родительские чаты, объявления терялись в общих группах, записи на мероприятия вели в таблицах.

**Решение:** Один Telegram-бот, который объединяет все коммуникации: оценки приходят автоматически, записи на мероприятия — в два клика, анонимный чат с психологом работает без дополнительных приложений.

## Стек

| Технология | Роль |
|---|---|
| [aiogram 3.x](https://docs.aiogram.dev/) | Telegram Bot Framework, FSM |
| SQLite | База данных |
| [Playwright](https://playwright.dev/) | Рендеринг HTML → PNG для табелей |
| APScheduler | Автоматическая рассылка табелей раз в 2 недели |
| openpyxl | Парсинг и генерация Excel-файлов |

## Быстрый старт

```bash
git clone https://github.com/reyssxxx/CardGen.git
cd CardGen
pip install -r requirements.txt
playwright install chromium
```

Создай `.env`:
```
BOT_TOKEN=your_bot_token
ADMIN_ID=123456789,987654321
```

Настрой `data/teachers.json`, `data/students.json`, `data/config.json` (см. ниже).

```bash
python main.py
```

## Конфигурация

**`data/teachers.json`** — учителя и их классы:
```json
{
  "telegram_user_id": {
    "name": "Иванова Анна Петровна",
    "subjects": ["Математика"],
    "classes": ["11Т", "11Г"]
  }
}
```

**`data/students.json`** — списки учеников по классам:
```json
{
  "11Т": ["Иванов Иван", "Петрова Мария"]
}
```

**`data/config.json`** — список предметов для табелей:
```json
{
  "subjects": ["Математика", "Физика", "Информатика"]
}
```

**`data/psychologists.json`** — психологи (не хранятся в БД):
```json
{
  "telegram_user_id": { "name": "Сидорова Елена Ивановна" }
}
```

```

│   └── states.py                  # FSM-состояния

├── database/                      # Репозитории (паттерн Repository)

│   └── scheduler_service.py       # APScheduler — авторассылка
```

## Формат Excel для загрузки оценок

| Строка | Содержимое |
|---|---|
| 1 | `Период:` \| `DD.MM.YYYY` (начало) \| `DD.MM.YYYY` (конец) |
| 2 | `Предмет` \| Ученик1 \| Ученик2 \| ... |
| 3+ | Название предмета \| оценки \| оценки \| ... |

Несколько оценок в ячейке разделяются пробелом: `4 5 3`. Допустимые значения: `2`, `3`, `4`, `5`, `н`, `б`.

## База данных

Файл `data/database.db` создаётся автоматически при первом запуске. При изменении схемы — удали файл и перезапусти бот.

Таблицы: `Users`, `Grades`, `Events`, `EventSections`, `EventRegistrations`, `Announcements`, `AnonQuestions`, `SupportChats`, `SupportMessages`.

## Роли пользователей

При `/start` роль определяется автоматически:

1. **Админ** — `user_id` в `ADMIN_ID` в `.env`
2. **Учитель** — `user_id` в `data/teachers.json`
3. **Психолог** — `user_id` в `data/psychologists.json`
4. **Зарегистрированный ученик** — есть в БД
5. **Новый пользователь** — проходит регистрацию (выбор класса и имени)
