# CardGen — Telegram-бот для лицея ЮФУ

Бот для автоматизации учёта оценок и внутренней коммуникации лицея Южного федерального университета (Ростов-на-Дону).

---

## Проблема

В лицее не было единого инструмента для взаимодействия между учениками, учителями и администрацией:

- Оценки выставлялись в журнале, но ученики не видели их в реальном времени — приходилось ждать или узнавать лично
- Учителя рассылали информацию вручную через общие чаты, где она терялась среди другого контента
- Запись на мероприятия велась в таблицах или устно — ни ученик, ни организатор не имели актуального списка
- Обратиться к психологу можно было только лично, что создавало барьер — особенно для тревожных тем
- Не было канала для анонимных вопросов к администрации

---

## Решение

Единый Telegram-бот, в который встроены все ключевые процессы лицея. Telegram выбран осознанно: он уже установлен у всех, не требует отдельной регистрации и работает на любом устройстве.

### Для учеников

**Табель оценок** — персональная карточка с оценками по всем предметам за каждый двухнедельный период. Формируется автоматически из данных учителей и приходит в личку по расписанию. Не нужно ждать конца четверти — прогресс виден в любой момент.

**Мероприятия** — лента предстоящих событий лицея с описанием, датой и секциями. Записаться можно в два нажатия, отменить — так же. Бот следит за лимитами мест и блокирует запись, если они исчерпаны.

**Объявления** — централизованная лента от администрации и учителей. Поддерживает текст и фото. Никаких общих чатов — только адресные сообщения.

**Вопросы к администрации** — анонимная форма с возможностью прикрепить фото. Ответ приходит в личку прямо в боте. Администратор видит автора вопроса, но ученик об этом не знает — снижает барьер для честной обратной связи.

**Психологическая поддержка** — анонимный чат с психологом внутри бота. Ученик начинает разговор без имени, и только если захочет — раскрывает личность. Работает как обычная переписка: без лишних подтверждений и статусов после каждого сообщения.

**Хендбук лицеиста** — справочник с информацией о лицее, правилах, расписании и типичных ситуациях. Открывается кнопкой прямо из главного меню.

---

### Для учителей

**Рассылка объявлений** — учитель выбирает один из своих классов и отправляет текст или фото всем его участникам. Никакого ручного копирования по чатам.

---

### Для администраторов

**Загрузка оценок через Excel** — учитель загружает файл в формате журнала, бот парсит его, показывает предпросмотр и после подтверждения сохраняет всё в базу. Исключает ошибки ручного ввода.

**Генерация табелей** — по запросу или автоматически раз в две недели бот формирует персональную карточку оценок для каждого ученика в виде изображения и отправляет в личку. Табель охватывает все периоды с сентября по текущий день.

**Управление оценками** — просмотр и удаление оценок по конкретному ученику, если была допущена ошибка при загрузке.

**Мероприятия** — создание событий с заголовком, датой, описанием и секциями (название, ведущий, время, вместимость). Бот следит за лимитами по классам и по секциям.

**Объявления** — рассылка всем ученикам или конкретному классу, с поддержкой фото.

**Q&A** — просмотр анонимных вопросов от учеников с возможностью ответить. Ответ автоматически пересылается автору вопроса.

---

### Для психолога

**Чат с учениками** — список активных анонимных обращений, переписка с каждым учеником, возможность завершить диалог. Бот ретранслирует сообщения между сторонами в реальном времени.

---

## Преимущества подхода

- **Один инструмент для всех** — ученик, учитель, администратор и психолог работают в одном боте, каждый видит только свой интерфейс
- **Не нужно ничего устанавливать** — Telegram есть у всех
- **Автоматизация рутины** — табели рассылаются сами, записи на мероприятия считаются автоматически, лимиты отслеживаются без участия человека
- **Анонимность там, где она важна** — психолог и Q&A спроектированы так, чтобы снизить барьер для обращения
- **Гибкая настройка** — классы, предметы, учителя и психологи настраиваются через JSON-файлы без изменения кода

## Стек

| Технология | Роль |
|---|---|
| Python 3.11+ | Основной язык |
| aiogram 3.x | Telegram Bot Framework, FSM |
| SQLite | База данных |
| Playwright | Рендеринг HTML → PNG для табелей оценок |
| APScheduler | Автоматическая рассылка раз в 2 недели |
| openpyxl | Парсинг и генерация Excel-файлов |

---

## Запуск

```bash
pip install -r requirements.txt
playwright install chromium
python main.py
```

Требуется файл `.env`:
```
BOT_TOKEN=your_bot_token
ADMIN_ID=123456789,987654321
```

И конфигурационные файлы в `data/`: `teachers.json`, `students.json`, `config.json`, `psychologists.json`.

## Структура проекта

```
├── main.py                         # Точка входа, регистрация роутеров
├── panel.py                        # Веб-панель управления
├── handlers/
│   ├── common_handlers.py          # /start, регистрация учеников
│   ├── admin/                      # Загрузка оценок, мероприятия, Q&A
│   ├── teacher_handlers.py         # Объявления от учителя
│   ├── psychologist_handlers.py    # Чат психолога
│   ├── student_handlers.py         # Табель, мероприятия, вопросы
│   ├── student_support_handlers.py # Анонимный чат ученика
│   └── states.py                   # FSM-состояния
├── keyboards/                      # Inline и Reply клавиатуры
├── database/                       # Репозитории (паттерн Repository)
├── services/
│   ├── excel_import_service.py     # Парсинг Excel с оценками
│   ├── grade_card_service.py       # Генерация PNG-табеля через Playwright
│   ├── mailing_service.py          # Рассылка табелей и объявлений
│   └── scheduler_service.py        # Автоматическая рассылка по расписанию
└── utils/                          # Валидаторы, пагинация, загрузка конфигов
```
